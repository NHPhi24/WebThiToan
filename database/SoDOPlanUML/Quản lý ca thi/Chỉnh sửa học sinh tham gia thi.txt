@startuml
autonumber
skinparam style strictuml
skinparam sequenceMessageAlign center

actor "Giáo viên" as Actor
boundary "Giao diện (UI)" as UI
control "Hệ thống" as System
database "DB" as DB

title Sơ đồ tuần tự: Chỉnh sửa học sinh tham gia thi

== Kiểm tra điều kiện ca thi ==
Actor -> UI: Chọn "Quản lý ca thi"
UI -> System: Yêu cầu lấy danh sách ca thi
System -> DB: Truy vấn danh sách
DB --> System: Trả kết quả dữ liệu
System --> UI: Hiển thị danh sách ca thi

Actor -> UI: Chọn "Xem chi tiết" ca thi
UI -> System: Gửi yêu cầu chi tiết kèm ID
System -> DB: Kiểm tra trạng thái ca thi
DB --> System: Trả về trạng thái

alt trạng thái là "Sẵn sàng"
    System --> UI: Hiển thị nút "Đăng ký thi"
    Actor -> UI: Chọn "Đăng ký thi"
    UI -> System: Yêu cầu danh sách học sinh đăng ký
    System -> DB: Truy vấn danh sách học sinh trong ca thi
    DB --> System: Trả về danh sách
    System --> UI: Hiển thị danh sách & nút "Chỉnh sửa"
    
    == Thực hiện chỉnh sửa ==
    Actor -> UI: Chọn "Chỉnh sửa" (Học sinh cụ thể)
    UI -> System: Yêu cầu Form chỉnh sửa thông tin
    System --> UI: Hiển thị Form kèm dữ liệu cũ
    
    Actor -> UI: Sửa thông tin (Tên, Email...) và nhấn "Lưu"
    UI -> System: Gửi yêu cầu cập nhật thông tin
    System -> DB: Kiểm tra Tên đăng nhập và Email đã tồn tại?
    DB --> System: Kết quả kiểm tra
    
    alt không tồn tại (Hợp lệ)
        System -> DB: Cập nhật thông tin vào CSDL (UPDATE)
        DB --> System: Xác nhận thành công
        System --> UI: Thông báo thành công
        UI --> Actor: Hiển thị thông báo thành công
    else đã tồn tại
        System --> UI: Thông báo thất bại (Tên đăng nhập/Email đã tồn tại)
        UI --> Actor: Hiển thị thông báo thất bại
    end
    
else trạng thái khác
    System --> UI: Chỉ hiển thị thông tin chi tiết
    UI --> Actor: Hiển thị giao diện xem thông tin (Không có quyền sửa)
end
@enduml
